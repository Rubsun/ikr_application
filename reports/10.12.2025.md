# Отчет по Code Review пакетов проекта

## Требования из readme.md

### Архитектура:
- **ui** - слой логики отображения данных
- **domain** - слой рабочей логики приложения
- **data** - слой для получения "сырых" данных или dto от внешнего API или из DB
- **Каждый слой знает только про тот слой, который находится ниже**

### Требования к проекту:
1. Создать свой "пакет" и добавить туда три подпакета: **data**, **domain**, **ui**
2. В своем пакете, в data, создать **Repository** который пока будет генерить фэйковые данные
3. В своем пакете, в domain, создать несколько **UseCase** которые должны будут из Repository брать данные и преобразовывать к обычным данным
4. В своем пакете, в ui, создать **Fragment**:
   - Добавить на него **ViewModel**, который будет получать данные из UseCase-ов
   - Добавить в него верстку (можно Compose, можно xml), которая отображает данные из ViewModel
   - Настроить маппинг данных от ViewModel к верстке
5. При создании верстки - учесть темы и стили

### Штрафы
1. Нарушение каждого пункта из требований - 2 балла (за каждый пункт)
2. Использование чужиж класса или верстки - 2 балла (за каждый юнит)
3. Создание лишних файлов - 1 балл (за каждый юнит)
4. Копипаста файла (полное совпадение структуры файла) - 1 балл (за каждый юнит)
